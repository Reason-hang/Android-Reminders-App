## 强提醒（最关键）

| 场景 | 预期结果 | 实测结果 | 备注 |
|---|---|---|---|
| App 完全在后台（近期用过）、到点提醒 | 响铃+震动+亮屏+全屏页 | | |
| App 被清后台（划掉最近任务）、到点提醒 | 响铃+震动+亮屏+全屏页 | | |
| 锁屏状态、到点提醒 | 响铃+震动+亮屏+全屏页+**锁屏能看到通知预览标题/备注**（本轮问题 2） | | |
| 提前 5 分钟提醒（本轮问题 1） | 提前 5 分钟：响铃+震动+亮屏+全屏页 | | |
| 到点提醒（提前提醒之后 5 分钟） | 再次：响铃+震动+亮屏+全屏页 | | |
| 查看系统闹钟调度 | `dumpsys alarm` 中同一 reminderId 同时存在 advance 和 due 两个不同 requestCode | | |
| 模拟前台服务启动失败 | 备用渠道仍按单条设置响铃/震动，并显示公开锁屏预览 | | |
| 锁屏查看通知来源 | 看到的是用户强提醒通知，不是低重要性的“提醒运行服务”通知 | | |
| 全屏页点击"关闭" | 停止响铃/震动 | | |
| 亮屏正常使用时到点 | 响铃+震动，并主动拉起全屏页；若系统拦截，至少显示高优先级横幅和通知记录 | | |
| 横幅或全屏页点击"关闭" | 停止响铃/震动和当前打断，通知中心仍保留可预览记录 | | |
| 两条不同提醒同一分钟触发 | 后一条正常强提醒；前一条转为通知记录；关闭前一条不能停止后一条 | | |
| 全屏页刚出现立即点"关闭" | 停止当前提醒并保留正确通知，不误操作上一条提醒 | | |
| 系统“闹钟音量”为 0 | App 日志明确记录音量为 0；调高闹钟音量后能正常响铃 | | |
| 全屏页点击"标为完成" | 非重复提醒：直接完成；重复提醒：**仅完成本次，不影响下一次**（2026-07-10 第二轮修复） | | |
| 通知栏点击"标为完成"按钮 | 同上，非重复直接完成；重复提醒仅完成本次 | | |
| 全屏页点击"稍后提醒 10 分钟" | 10 分钟后重新触发 | | |
| 全屏页点击"标为完成" | 列表同步更新为已完成 | | |
