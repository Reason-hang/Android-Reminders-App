## 核心场景

| 编号 | 场景 | 预期 |
|---|---|---|
| A1 | 锁屏正式到点 | 亮屏、声音、震动、全屏页和锁屏标题备注均可见 |
| A2 | App 后台正式到点 | 声音、震动和业务通知正常 |
| A3 | 划掉最近任务后到点 | 仍能触发完整提醒 |
| A4 | 已解锁使用手机时到点 | 声音、震动，系统允许时全屏，否则高优先级横幅 |
| A5 | 提前提醒 | 提前事件独立完整提醒 |
| A6 | 提前后正式到点 | 正式事件再次完整提醒，不能被提前事件覆盖 |
| A7-A9 | 声音/震动开关组合 | 只执行用户开启的媒介，仍保留视觉提醒 |
| A10-A11 | 关闭或划除 | 停止当前打断，保留静音通知记录 |
| A12-A13 | 稍后 10 分钟 | 当前通知结束，稍后再次提醒且不跳过重复周期 |
| A14 | 完成 | 一次性提醒完成；重复提醒只完成本次 |

## 恢复与并发

- 两条提醒同一分钟：后一条成为当前打断，前一条保留通知记录。
- 点击前一条旧操作：不得停止后一条。
- 同一重复提醒的 ADVANCE 与 DUE 连续到达：DUE 正常重新提醒。
- 重复投递同一个 PendingIntent：occurrence 只能推进一次。
- 重启、覆盖安装、精确闹钟权限恢复后：未来提醒重新注册，单条失败不阻断其他提醒。
- v1.3 覆盖安装 v1.4：旧提醒、分类、重复和提前设置保留，数据库无崩溃。

## 采证命令

```bash
ADB=/opt/homebrew/share/android-commandlinetools/platform-tools/adb
$ADB devices -l
$ADB shell getprop ro.product.model
$ADB shell getprop ro.build.version.release
$ADB shell getprop ro.build.version.sdk
$ADB shell getprop ro.build.display.id
$ADB shell dumpsys package com.reminder.local | grep -E 'versionName|versionCode'
```

触发前清空日志，并分别保存完整日志和过滤日志。重点观察 `AlarmScheduler`、`AlarmReceiver`、`AlarmAlertService`、`NotificationActionReceiver`、`ActivityTaskManager` 和 `NotificationService`。
