## 产品与架构不变量

- `ADVANCE`、`DUE`、`SNOOZE` 独立；只有 `DUE` 推进重复周期。
- 每个操作携带 `reminderId + alarmId + kind + occurrenceTime`。
- 旧 occurrence 只能结束自身打断，不能停止或修改新 occurrence。
- `triggerTime` 是重复模板，`nextTriggerTime` 是实际下一次时间。
- 所有重复规则按设备当前时区计算。
- Room 与 AlarmManager 采用补偿式一致性，不假设跨系统 ACID 事务。
- 强提醒验收链为：Alarm 注册 → Receiver → 前台服务 → 声音/震动 → WakeLock → 全屏页 → 锁屏业务通知。
- 不增加 `INTERNET`，不提交 APK、keystore、密码、token 或 `local.properties`。
